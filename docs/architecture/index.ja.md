# アーキテクチャ概要

Network Simulator は、ブラウザ上でネットワークトポロジーを視覚的に設計し、実際の Docker コンテナとしてデプロイできる Web ベースの学習環境です。設計思想は **「本物のネットワーク、ゼロ手間」** ：各ルーター・スイッチ・ターミナルは、実際のルーティングソフトウェア（FRR）を動かすライブコンテナであり、リアルな仮想イーサネット（veth）ペアで接続され、ブラウザ内のターミナルからアクセス可能です。

このシステムは **ステートレスなバックエンド** を中心に設計されています。React フロントエンドがすべてのトポロジー設定状態を保持し、デプロイや設定変更のたびにトポロジー全体をバックエンドへ送信します。バックエンドは純粋な実行エンジンであり、設定ファイルのレンダリング・Containerlab の呼び出し・`docker exec` コマンドの実行・結果の返却のみを担います。フロントエンドが常に唯一の信頼できる情報源となることで、トポロジー全体を再デプロイすることなく、個別ノードへの部分的な再設定が可能です。

重要なアーキテクチャ機能として **ゼロダウンタイム設定変更** があります。OSPF・RIP・BGP・スタティックルート・IP アドレスなどのルーター設定は、デーモンを再起動せずに実行中のコンテナへ適用できます。これは FRR の `frr-reload.py` ツールを使用することで実現しており、旧設定と新設定を差分比較し、変更点のみを `vtysh` 経由で適用します。
