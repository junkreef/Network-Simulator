# ユーザーガイド

> 🇬🇧 English version available → [user-guide.md](./user-guide.md)

このガイドでは、Network Simulator を使ってコンテナ化されたネットワークトポロジーを構築・設定・実行する方法を説明します。

---

## 画面構成

アプリケーションは 4 つの主要エリアで構成されています:

| エリア | 場所 | 役割 |
|---|---|---|
| **ヘッダー（ツールバー）** | 上部バー | デプロイ / 破棄ボタン、グローバル操作 |
| **キャンバス** | 中央（メインエリア） | トポロジーノードの配置・接続 |
| **プロパティパネル** | 右サイドバー | 選択ノードの設定 |
| **Web ターミナル** | 下部パネル | 実行中コンテナへのブラウザ内シェル |

---

## ノードの追加

ノードはネットワーク機器（物理または仮想）を表します。3 種類のノードタイプがあります:

- **Router** — OSPF / RIP / BGP をサポートする FRR ベースの Alpine コンテナ
- **L2 Switch** — Linux VLAN ブリッジを持つ Alpine コンテナ
- **Terminal** — エンドホストとして機能するシンプルな Alpine コンテナ

ノードを追加するには、左サイドバーの**ノードパレットからキャンバスにドラッグ**します。キャンバスの空きエリアを右クリックして表示されるコンテキストメニューからも追加できます。

各ノードには名前とステータスバッジ（デプロイ後に表示）が表示されます。

---

## ノードの接続

接続はデバイス間のインターフェースを繋ぐ物理ケーブルを表します。

1. ノードにカーソルを合わせると、接続**ハンドル**（小さな点）がエッジに表示されます。
2. ハンドルをクリックしてドラッグし、別のノードのハンドルに接続します。
3. 作成されたエッジはポイントツーポイントリンク（コンテナ環境では veth ペア）を表します。

エッジを選択して `Delete` または `Backspace` を押すと削除できます。

---

## ルーターの設定

Router ノードを選択すると、プロパティパネルに設定画面が表示されます。

### インターフェース

各インターフェース（`eth1` ～ `eth4`）に IP アドレスを設定します:

- **CIDR 表記**で入力: `192.168.1.1/24`
- 接続されたエッジは接続順にインターフェースに割り当てられます。

### VLAN サブインターフェース

dot1q サブインターフェース（例: `eth1.10`）を作成します:

1. **サブインターフェースを追加** をクリック。
2. **親インターフェース**（例: `eth1`）、**VLAN ID**（例: `10`）、**IP アドレス**（CIDR）を設定。

### OSPF

1. **OSPF トグル**を有効にします。
2. **ルーター ID** を設定（例: `1.1.1.1`）。
3. **エリア**を追加:
   - **エリア ID**（例: バックボーンは `0`、スタブは `1` など）
   - **ネットワーク**（CIDR 形式、例: `192.168.1.0/24`）
   - **エリアタイプ**: `normal`、`stub`、`totally-stub`、`nssa`、`totally-nssa`
4. オプション — **ルート集約（range）**: エリア内ネットワークの集約。
5. オプション — **再配布（Redistribute）**: `connected`、`static`、`rip`、`bgp` のルートを OSPF に注入。
6. オプション — **default-information originate**: デフォルトルートを OSPF に広告。

### RIP

1. **RIP トグル**を有効にします。
2. RIP アドバタイズに含める**ネットワーク**（CIDR）を追加。
3. オプション — **再配布**: `connected`、`static`、`ospf`、`bgp` のルートを注入。

### BGP

1. **BGP トグル**を有効にします。
2. **AS 番号**と**ルーター ID** を設定。
3. **ネイバー**を追加:
   - ピアの **IP アドレス**
   - **リモート AS** 番号
4. オプション — **再配布**: `connected`、`static`、`ospf`、`rip` のルートを注入。

### スタティックルート

スタティックルートを追加します:

- **宛先**: CIDR 表記（例: `10.0.0.0/8`）
- **ネクストホップ**: ゲートウェイの IP アドレス（例: `192.168.1.254`）

### デフォルトゲートウェイ

このルーターのデフォルトゲートウェイの IP アドレスを入力します。`0.0.0.0/0` スタティックルートとして設定されます。

---

## L2 スイッチの設定

L2 Switch ノードを選択してポートを設定します。各ポートは接続されたエッジに対応します。

| モード | 説明 |
|---|---|
| **None** | ポートは無効（VLAN 設定なし） |
| **Access** | 単一 VLAN のトラフィックを通過させる。**VLAN ID**（1–4094）を 1 つ設定 |
| **Trunk** | 複数 VLAN を通過させる。**VLAN ID** をカンマ区切りで設定 |

トランク VLAN の例: `10,20,30`

> 内部的には、Alpine コンテナ内で VLAN フィルタリングを有効にした Linux ブリッジ（`br0`）を使用しています。

---

## ターミナルの設定

Terminal ノードを選択して設定します:

- **IP アドレス** — CIDR 表記（例: `192.168.1.100/24`）
- **デフォルトゲートウェイ** — 上流ルーターのインターフェース IP（例: `192.168.1.1`）

## トポロジーのデプロイ

トポロジーが完成したら:

1. ヘッダーツールバーの **Deploy** をクリック。
2. フロントエンドがトポロジー全体の JSON を `POST /api/v1/topology/deploy` に送信。
3. バックエンドが処理:
   - Jinja2 テンプレートから `topology.clab.yml` を生成。
   - `containerlab deploy` を実行し、Docker コンテナと veth ペアを作成。
4. コンテナが起動すると、ノードのステータスバッジが **up**（緑）または **down**（赤）に更新されます。

> デプロイにより実際の Docker コンテナが作成されます。事前に Docker と Containerlab が起動していることを確認してください。

---

## ノード設定の適用

デプロイ後、ノード設定（IP アドレス、ルーティングプロトコル）は別途適用します:

1. キャンバスでノードを選択。
2. プロパティパネルの **Apply** をクリック。
3. バックエンドが**コンテナを再起動せずに**設定を適用:
   - **ルーター**: `frr.conf` を生成し、コンテナ内で `frr-reload.py` を実行。
   - **スイッチ**: シェルコマンドで Linux ブリッジと VLAN ポートマッピングを設定。
   - **ターミナル**: IP アドレスとデフォルトルートを設定。

トポロジーのデプロイ中はいつでも設定を変更して再適用できます。

---

## Web ターミナル

実行中のすべてのコンテナにブラウザ内ターミナルからアクセスできます:

1. ノード上の**ターミナルアイコン**（📟）をクリック。
2. 下部パネルに **Xterm.js** ターミナルが開き、コンテナ内の `/bin/sh` に接続されます。
3. インタラクティブなシェルに完全アクセスできます。

便利なコマンド:

```bash
# インターフェースの IP アドレスを確認
ip addr

# ルーティングテーブルを表示
ip route

# FRR ルーティング CLI を開く（ルーターのみ）
vtysh

# 別ノードに ping
ping 192.168.1.1

# ARP テーブルを表示
ip neigh
```

ターミナルを閉じるには、ターミナルパネルの **×** をクリックします。

---

## ランタイムステータスの確認

デプロイ済みのノードを選択すると、プロパティパネルでリアルタイムの情報を確認できます:

| タブ | 情報 |
|---|---|
| **ルーティングテーブル** | アクティブなルート（`ip route show`） |
| **ARP テーブル** | 隣接ノードの MAC/IP マッピング（`ip neigh`） |
| **OSPF ネイバー** | OSPF 隣接状態（`vtysh -c "show ip ospf neighbor"`） |
| **BGP サマリー** | BGP セッション状態とプレフィックス数 |
| **RIP ステータス** | RIP ルーティングテーブルとネイバー状態 |

---

## トポロジーの破棄

実行中のすべてのコンテナを削除してネットワークインターフェースをクリーンアップするには:

1. ヘッダーツールバーの **Destroy** をクリック。
2. バックエンドが `containerlab destroy` を呼び出し、すべてのコンテナと veth ペアを削除。
3. ノードのステータスバッジが **stopped** にリセットされます。

> トポロジーを破棄しても、キャンバスの設定は削除されません。いつでも再デプロイできます。
